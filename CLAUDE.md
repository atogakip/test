# CLAUDE.md

## このリポジトリでのClaude Codeの役割

GitHub IssueのURL を受け取り、受け入れ条件をもとにテストケースを日本語で生成する。

---

## テストケース生成のルール

### 入力
- GitHub Issue の URL（受け入れ条件が記載されているもの）

### 出力形式
以下の項目をCSV形式で出力すること。Googleスプレッドシートにそのまま貼り付けられるようにする。

| 項目 | 説明 |
|------|------|
| ID | TC-001 形式で連番 |
| テストケース名 | 何をテストするかを一言で |
| テスト観点 | 正常系 / 異常系 / 境界値 のいずれか |
| 前提条件 | テスト実施前に満たしておく状態 |
| 操作手順 | 番号付きで具体的に記述 |
| 期待結果 | 操作後に起きるべきこと |
| 実行結果 | 空欄（QAが手動で記入する） |
| 備考 | 特記事項があれば記載、なければ空欄 |

### 生成のガイドライン
- 受け入れ条件の1項目につき、最低でも「正常系」「異常系」の2ケースを生成する
- 操作手順は誰が読んでも迷わないよう具体的に書く
- 専門用語は避け、QAメンバーが理解できる言葉で書く
- 出力はCSV形式のみとし、余計な説明文は不要

---

## 実行方法

Claude Code のチャットで以下のように入力する：

```
下記IssueのテストケースをCSV形式で生成してください。
[GitHub IssueのURL]
```

---

## 出力例

```csv
ID,テストケース名,テスト観点,前提条件,操作手順,期待結果,実行結果,備考
TC-001,正常なメールアドレスでログインできる,正常系,ユーザーアカウントが作成済みであること,"1. ログインページを開く
2. 正しいメールアドレスを入力する
3. 正しいパスワードを入力する
4. ログインボタンをクリックする",ダッシュボードページに遷移する,,
TC-002,誤ったパスワードでログインできない,異常系,ユーザーアカウントが作成済みであること,"1. ログインページを開く
2. 正しいメールアドレスを入力する
3. 誤ったパスワードを入力する
4. ログインボタンをクリックする",「パスワードが正しくありません」のエラーメッセージが表示される,,
```
